# Local API Specification (V1)

## Purpose

This document defines the V1 contract between the frontend and the local backend.
The goal is to keep the frontend independent from filesystem, SQLite, and other local implementation details.

## Resolved Decisions

1. The V1 boundary is an HTTP JSON API.
2. The API base path is `/api/v1`.
3. The frontend never reads Chrome bookmark files or SQLite directly.
4. A bookmark item has exactly one current summary version.
5. A summary is generated by AI and can then be edited manually by the user.
6. AI tag suggestions are not persisted automatically; the user must confirm them.
7. Bookmark sync is asynchronous, and only one sync run may be active at a time.
8. Settings responses never return the raw OpenRouter API key.

## Conventions

### Transport

- Protocol: HTTP
- Content type: `application/json`
- Base path: `/api/v1`

### Data Rules

- All timestamps use ISO 8601 in UTC, for example `2026-04-08T19:00:00Z`
- JSON fields use `camelCase`
- Resource IDs are opaque strings
- Enum values are lowercase strings

### Error Format

All non-2xx responses use this shape:

```json
{
  "error": {
    "code": "validation_error",
    "message": "Status value is not supported.",
    "details": {
      "field": "status"
    }
  }
}
```

Common error codes:

- `validation_error`
- `not_found`
- `conflict`
- `ai_not_configured`
- `sync_already_running`
- `upstream_error`

Frontend behavior:

- The API returns structured JSON errors for machine handling.
- The web client renders user-readable messages instead of raw JSON payloads.
- For `upstream_error` from AI endpoints, the web client should include guidance to check the OpenRouter API key, model name, network access, or provider availability.
- If the backend provides more specific OpenRouter guidance, such as a rate limit message, the web client should render that specific message without adding generic guidance.

## Resource Models

### Item

```json
{
  "id": "itm_01JQXYZ123",
  "url": "https://example.com/article",
  "normalizedUrl": "https://example.com/article",
  "title": "Example Article",
  "domain": "example.com",
  "status": "new",
  "importedAt": "2026-04-08T18:00:00Z",
  "updatedAt": "2026-04-08T18:00:00Z",
  "tags": [
    {
      "id": "tag_01JQTAG001",
      "name": "frontend"
    }
  ],
  "summary": {
    "content": "Short article summary.",
    "updatedAt": "2026-04-08T18:10:00Z",
    "updatedBy": "ai"
  }
}
```

Field rules:

- `status`: `new`, `read`, or `archived`
- `summary`: `null` when no summary exists yet
- `summary.updatedBy`: `ai` or `user`

### Tag

```json
{
  "id": "tag_01JQTAG001",
  "name": "frontend",
  "createdAt": "2026-04-08T18:05:00Z",
  "updatedAt": "2026-04-08T18:05:00Z"
}
```

Tag rules:

- Tag names are unique case-insensitively in V1
- Leading and trailing whitespace is trimmed before save

### Summary

```json
{
  "itemId": "itm_01JQXYZ123",
  "content": "Short article summary.",
  "updatedAt": "2026-04-08T18:10:00Z",
  "updatedBy": "ai"
}
```

Summary rules:

- Only one current summary is stored per item
- AI generation replaces the current stored summary
- Manual editing updates the same stored summary
- AI-generated summaries are requested in Russian for V1.
- Summary version history is not stored in V1

### Settings

```json
{
  "openRouter": {
    "apiKeyConfigured": true,
    "model": "google/gemma-4-31b-it:free",
    "summaryPrompt": "Summarize a bookmarked web page for a personal reading inbox. Use only the provided metadata and page signals. Do not invent missing facts. If information is limited, provide a cautious short summary and explicitly mention uncertainty. Write in Russian. Keep the response within 5 sentences. Return only the summary text.",
    "tagsPrompt": "Suggest 3 to 5 short lowercase tags for a bookmarked web page. Use only the provided context and do not invent missing facts. Return one tag per line. Use Russian when it fits the page topic."
  }
}
```

Settings rules:

- `GET /settings` never returns the raw API key
- `PATCH /settings` may set or clear the API key
- `model` is one shared model setting used for both summary generation and tag suggestion.
- `summaryPrompt` and `tagsPrompt` are user-editable prompt templates for summary and tag generation.
- If `summaryPrompt` or `tagsPrompt` is empty or missing, backend uses built-in defaults.
- If `model` is missing, backend falls back to ranked default from product model table.

### Sync Status

```json
{
  "status": "succeeded",
  "startedAt": "2026-04-08T18:20:00Z",
  "finishedAt": "2026-04-08T18:20:03Z",
  "importedCount": 12,
  "updatedCount": 3,
  "skippedCount": 40,
  "error": null
}
```

Sync status values:

- `idle`
- `running`
- `succeeded`
- `failed`

Sync rules:

- Sync deduplicates items by `normalizedUrl`
- If a bookmark with the same `normalizedUrl` already exists, the backend updates the existing item instead of creating a duplicate
- Removing a bookmark from Chrome does not delete the local item in V1
- Only one sync run can be active at a time

## Endpoints

### `GET /health`

Returns backend availability.

Response `200 OK`:

```json
{
  "status": "ok"
}
```

### `GET /items`

Returns bookmark items for the inbox and search views.

Query parameters:

- `q`: full-text search across `title`, `url`, `domain`, and `summary.content`
- `status`: one of `new`, `read`, `archived`
- `tagIds`: comma-separated tag IDs
- `limit`: default `50`, max `200`
- `offset`: default `0`
- `sort`: `importedAt:desc`, `importedAt:asc`, `updatedAt:desc`, or `updatedAt:asc`

Response `200 OK`:

```json
{
  "items": [
    {
      "id": "itm_01JQXYZ123",
      "url": "https://example.com/article",
      "normalizedUrl": "https://example.com/article",
      "title": "Example Article",
      "domain": "example.com",
      "status": "new",
      "importedAt": "2026-04-08T18:00:00Z",
      "updatedAt": "2026-04-08T18:10:00Z",
      "tags": [
        {
          "id": "tag_01JQTAG001",
          "name": "frontend"
        }
      ],
      "summary": {
        "content": "Short article summary.",
        "updatedAt": "2026-04-08T18:10:00Z",
        "updatedBy": "user"
      }
    }
  ],
  "total": 1,
  "limit": 50,
  "offset": 0
}
```

### `GET /items/:itemId`

Returns one item with its current summary and tags.

Response `200 OK`:

```json
{
  "id": "itm_01JQXYZ123",
  "url": "https://example.com/article",
  "normalizedUrl": "https://example.com/article",
  "title": "Example Article",
  "domain": "example.com",
  "status": "new",
  "importedAt": "2026-04-08T18:00:00Z",
  "updatedAt": "2026-04-08T18:10:00Z",
  "tags": [
    {
      "id": "tag_01JQTAG001",
      "name": "frontend"
    }
  ],
  "summary": {
    "content": "Short article summary.",
    "updatedAt": "2026-04-08T18:10:00Z",
    "updatedBy": "user"
  }
}
```

Errors:

- `404 Not Found` when the item does not exist

### `PATCH /items/:itemId`

Updates mutable item fields.

Request body:

```json
{
  "status": "read"
}
```

Response `200 OK`:

```json
{
  "id": "itm_01JQXYZ123",
  "url": "https://example.com/article",
  "normalizedUrl": "https://example.com/article",
  "title": "Example Article",
  "domain": "example.com",
  "status": "read",
  "importedAt": "2026-04-08T18:00:00Z",
  "updatedAt": "2026-04-08T18:12:00Z",
  "tags": [],
  "summary": null
}
```

Rules:

- In V1, only `status` is mutable through this endpoint
- Unsupported fields return `400 Bad Request`

### `GET /tags`

Returns all tags.

Response `200 OK`:

```json
{
  "tags": [
    {
      "id": "tag_01JQTAG001",
      "name": "frontend",
      "createdAt": "2026-04-08T18:05:00Z",
      "updatedAt": "2026-04-08T18:05:00Z"
    }
  ]
}
```

### `POST /tags`

Creates a new tag.

Request body:

```json
{
  "name": "frontend"
}
```

Response `201 Created`:

```json
{
  "id": "tag_01JQTAG001",
  "name": "frontend",
  "createdAt": "2026-04-08T18:05:00Z",
  "updatedAt": "2026-04-08T18:05:00Z"
}
```

Errors:

- `409 Conflict` if a tag with the same normalized name already exists

### `PATCH /tags/:tagId`

Renames a tag.

Request body:

```json
{
  "name": "web-frontend"
}
```

Response `200 OK`:

```json
{
  "id": "tag_01JQTAG001",
  "name": "web-frontend",
  "createdAt": "2026-04-08T18:05:00Z",
  "updatedAt": "2026-04-08T18:15:00Z"
}
```

### `DELETE /tags/:tagId`

Deletes the tag and removes its associations from all items.

Response `204 No Content`

### `POST /items/:itemId/tags`

Attaches an existing tag to an item.

Request body:

```json
{
  "tagId": "tag_01JQTAG001"
}
```

Response `200 OK`:

```json
{
  "id": "itm_01JQXYZ123",
  "tags": [
    {
      "id": "tag_01JQTAG001",
      "name": "frontend"
    }
  ]
}
```

Rules:

- Re-attaching an already attached tag is idempotent and returns `200 OK`

### `DELETE /items/:itemId/tags/:tagId`

Removes a tag from an item.

Response `204 No Content`

### `GET /items/:itemId/summary`

Returns the current stored summary.

Response `200 OK`:

```json
{
  "itemId": "itm_01JQXYZ123",
  "content": "Short article summary.",
  "updatedAt": "2026-04-08T18:10:00Z",
  "updatedBy": "user"
}
```

Errors:

- `404 Not Found` when no summary exists yet

### `POST /items/:itemId/summary`

Generates or regenerates the current summary with AI.
The backend builds a cleaned metadata context from stored item fields and best-effort lightweight page signals:
- stored fields: `title`, `url`, `domain`, existing summary (when present);
- page signals (when reachable): `meta description`, `og:title`, `og:description`, `keywords`, `author`, published time fields, plus page `h1`/`h2` headings.

The backend does not run full article-content extraction in this flow; metadata/signal enrichment is best-effort and does not create a dedicated failure class when unavailable.

Request body:

```json
{}
```

Response `200 OK`:

```json
{
  "itemId": "itm_01JQXYZ123",
  "content": "Short article summary generated by AI.",
  "updatedAt": "2026-04-08T18:10:00Z",
  "updatedBy": "ai"
}
```

Errors:

- `404 Not Found` when the item does not exist
- `409 Conflict` with `ai_not_configured` when OpenRouter is not configured
- `502 Bad Gateway` with `upstream_error` when AI generation fails upstream
- `upstream_error` may use a specific message for OpenRouter rate limits, rejected keys, rejected model names, or invalid request formats

### `PATCH /items/:itemId/summary`

Updates the current stored summary manually.

Request body:

```json
{
  "content": "Refined summary edited by the user."
}
```

Response `200 OK`:

```json
{
  "itemId": "itm_01JQXYZ123",
  "content": "Refined summary edited by the user.",
  "updatedAt": "2026-04-08T18:11:00Z",
  "updatedBy": "user"
}
```

Rules:

- This endpoint requires an existing summary
- Manual editing overwrites the current stored summary

Errors:

- `404 Not Found` when no summary exists yet

### `POST /items/:itemId/tag-suggestions`

Generates AI tag suggestions for an item.

Request body:

```json
{}
```

Response `200 OK`:

```json
{
  "itemId": "itm_01JQXYZ123",
  "suggestions": [
    "frontend",
    "performance",
    "web"
  ]
}
```

Rules:

- Suggestions are not saved automatically
- The frontend must explicitly attach confirmed tags through tag endpoints
- If an item already has a stored summary, tag suggestions are generated from that summary context first.
- If summary is missing, tag suggestions are generated from cleaned item metadata context enriched with the same best-effort page signals used by summary generation (meta/og fields and `h1`/`h2` headings), without persisting a new summary.
- The backend asks OpenRouter for one short tag per line and tolerates JSON, comma-separated, or newline-separated model output.

Errors:

- `404 Not Found` when the item does not exist
- `409 Conflict` with `ai_not_configured` when OpenRouter is not configured
- `502 Bad Gateway` with `upstream_error` when AI generation fails upstream

### `POST /sync/bookmarks`

Starts bookmark import from the local Chrome profile.

Response `202 Accepted`:

```json
{
  "status": "running",
  "startedAt": "2026-04-08T18:20:00Z",
  "finishedAt": null,
  "importedCount": 0,
  "updatedCount": 0,
  "skippedCount": 0,
  "error": null
}
```

Errors:

- `409 Conflict` with `sync_already_running` when a sync is already active

### `GET /sync/status`

Returns the latest sync state.

Response `200 OK`:

```json
{
  "status": "succeeded",
  "startedAt": "2026-04-08T18:20:00Z",
  "finishedAt": "2026-04-08T18:20:03Z",
  "importedCount": 12,
  "updatedCount": 3,
  "skippedCount": 40,
  "error": null
}
```

Updates application settings.

Request body:

```json
{
  "openRouter": {
    "apiKey": "or-v1-xxxxx",
    "model": "google/gemma-4-31b-it:free",
    "summaryPrompt": "Summarize a bookmarked web page for a personal reading inbox. Use only the provided metadata and page signals. Do not invent missing facts. If information is limited, provide a cautious short summary and explicitly mention uncertainty. Write in Russian. Keep the response within 5 sentences. Return only the summary text.",
    "tagsPrompt": "Suggest 3 to 5 short lowercase tags for a bookmarked web page. Use only the provided context and do not invent missing facts. Return one tag per line. Use Russian when it fits the page topic."
  }
}
```

Response `200 OK`:

```json
{
  "openRouter": {
    "apiKeyConfigured": true,
    "model": "google/gemma-4-31b-it:free",
    "summaryPrompt": "Summarize a bookmarked web page for a personal reading inbox. Use only the provided metadata and page signals. Do not invent missing facts. If information is limited, provide a cautious short summary and explicitly mention uncertainty. Write in Russian. Keep the response within 5 sentences. Return only the summary text.",
    "tagsPrompt": "Suggest 3 to 5 short lowercase tags for a bookmarked web page. Use only the provided context and do not invent missing facts. Return one tag per line. Use Russian when it fits the page topic."
  }
}
```

Rules:

- `openRouter.apiKey` is optional in patch requests
- Sending an empty string for `openRouter.apiKey` clears the saved key
- `openRouter.model` controls both summary and tag generation model choice.
- `openRouter.summaryPrompt` controls summary-generation instruction template.
- `openRouter.tagsPrompt` controls tag-suggestion instruction template.
- Sending an empty string for `openRouter.summaryPrompt` or `openRouter.tagsPrompt` resets that prompt to backend default.
- The app remains usable without OpenRouter configuration; only AI endpoints fail

## Out of Scope for V1

- Chrome bookmark editing through the API
- Summary history or revision tracking
- Automatic persistence of AI tag suggestions
- Multi-user authentication or authorization
- Cloud-only API behavior
